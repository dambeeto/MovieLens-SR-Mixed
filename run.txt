== MovieLens-SR-Mixed -- pelny pipeline: dane + KMeans + sentyment + rekomendacje + popularnosc + bonusy + FastAPI ==

WYMAGANIA: Python 3.10+, ok. 2 GB wolnego miejsca na dysku (rozpakowane dane ~875 MB + parquet).

1. Setup srodowiska
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1        # PowerShell (Windows)
   # source .venv/bin/activate         # bash (Linux/Mac)
   python -m pip install --upgrade pip
   python -m pip install -r requirements.txt

2. Pobranie danych (ml-1m + ml-20m z files.grouplens.org)
   python scripts/01_download.py
   # ~5 min na 20M; 1M jest natychmiastowy. Skrypt idempotentny -- pomija jesli pliki juz sa.

3. Czyszczenie, identyfikacja brakow, podzial train/test, zapis parquet
   python scripts/02_clean.py
   # Wynik:
   #   data/processed/{users_1m,movies_1m,ratings_1m,ratings_20m,tags_20m}.parquet
   #   data/processed/{train,test}.parquet (chronologiczny split 80/20 na ratings_1m)
   #   reports/missing_report.csv

4. Analiza wstepna (EDA) -- wykresy
   python scripts/03_eda.py
   # Wynik: reports/ratings_distribution.png, users_demographics.png,
   #        genres_popularity.png, ratings_over_time.png

5. Budowa cech uzytkownikow (macierz user x feature)
   python scripts/04_features.py
   # Wynik: data/processed/user_features.parquet (~6040 userow x ~59 cech)

6. Trening KMeans + interpretacja klastrow
   python scripts/05_cluster.py
   # Wynik:
   #   models/{scaler,kmeans}.joblib
   #   reports/k_selection.csv (silhouette + davies_bouldin dla k=2..15)
   #   reports/clusters_pca.png
   #   reports/cluster_profiles.csv
   #   data/processed/user_clusters.parquet (user_id -> cluster_id)
   #   log w konsoli z opisem kazdego klastra

7. Klasyfikacja sentymentu tagow (ml-20m)
   python scripts/06_sentiment.py
   # Pipeline:
   #   1. Filtrowanie tagow -- zostawia tylko tagi z przymiotnikami/przyslowkami (NLTK POS)
   #   2. Etykietowanie -- VADER nadaje etykiety positive/negative (neutralne usuniete)
   #   3. Trening -- TF-IDF (5000 terms) + Logistic Regression
   # Wynik:
   #   models/sentiment_vectorizer.joblib
   #   models/sentiment_clf.joblib
   #   log w konsoli z classification report (accuracy, precision, recall, F1)
   #
   # Statystyki: 465_541 tagow -> 117_365 po filtrze POS -> 44_272 po usunieciu neutralnych
   # Wyniki: accuracy=0.99, F1 macro=0.98

8a. System rekomendacji filmow (collaborative filtering na ml-1m)
   python scripts/07_recsys.py
   # Wymaga: data/processed/{train,test,movies_1m}.parquet (powstaja w kroku 3).
   # Pipeline:
   #   1. Fituje trzy modele na train.parquet:
   #      - SVD (scipy.sparse.linalg.svds, k=50 latent factors, mean-centered)
   #      - ItemKNN-cosine (k=30 sasiadow)
   #      - ItemKNN-pearson (k=30 sasiadow, similarity na user-mean-centered)
   #   2. Ewaluuje na test.parquet: RMSE, MAE, Hit-rate@10 (sample 1000 userow, prog like>=4.0)
   #   3. Zapisuje top-10 dla 5 losowych userow per model
   # Wynik:
   #   models/recsys/{svd, item_knn_cosine, item_knn_pearson}.joblib
   #   models/recsys/{user_index, item_index}.joblib
   #   reports/recsys/metrics.csv                        (RMSE/MAE/Hit-rate@10 per model)
   #   reports/recsys/similarity_comparison.csv          (cosine vs pearson side-by-side)
   #   reports/recsys/top10_examples.csv                 (5 userow x 3 modele x 10 filmow)
   #   reports/recsys/rating_predictions_scatter.png     (pred vs actual dla najlepszego modelu)
   #   log w konsoli z porownaniem modeli + przykladowa top-10 listy filmow

8b. Prognoza popularnosci gatunkow (regresja czasowa na ml-20m)
   python scripts/08_popularity.py
   # Wymaga: data/processed/ratings_20m.parquet (powstaje w kroku 3) + ml-20m/movies.csv (kr. 2).
   # Pipeline:
   #   1. Agregacja po (gatunek, kwartal) z multi-genre filmow z 20m -- ~20 gatunkow x ~80 kwartalow
   #   2. Drop niepelnego ostatniego kwartalu (ml-20m konczy sie mid-Q1 2015)
   #   3. Features: log1p(n_ratings), sezonowosc sin/cos, lagi (1,2,4), rolling mean 4
   #   4. Trening + walk-forward eval na ostatnich 4 kwartalach: LinearRegression vs Ridge vs RandomForest
   #   5. Prognoza next-quarter dla wszystkich gatunkow
   # Wynik:
   #   models/popularity/{best_model, history, meta}.joblib/parquet
   #   reports/popularity/model_comparison.csv         (RMSE-log / MAE / MAPE per model)
   #   reports/popularity/forecast_next.csv            (prognoza dla kazdego gatunku)
   #   reports/popularity/time_series_per_genre.png    (top-8 gatunkow + zaznaczona prognoza)

9. Bonusy: identyfikacja outlierow + selekcja atrybutow
   python scripts/09_bonus.py
   # Wymaga: data/processed/user_features.parquet (krok 5) + user_clusters.parquet (krok 6).
   # Wynik:
   #   data/processed/user_outliers.parquet
   #   models/bonus/{isolation_forest, isolation_scaler}.joblib
   #   reports/bonus/outliers_pca.png                  (scatter z czerwonymi outlierami)
   #   reports/bonus/top_anomalous_users.csv           (top-10 najbardziej anomalnych)
   #   reports/bonus/variance_report.csv               (per-feature variance, low_variance flag)
   #   reports/bonus/mutual_info_top20.csv             (top-20 cech vs cluster_id)

10. Aplikacja webowa (FastAPI + UI)
   uvicorn main:app --reload
   # Domyslnie http://127.0.0.1:8000
   # Endpointy:
   #   GET  /                       -> single-page UI (static/index.html)
   #   GET  /api/health             -> flagi dostepnosci modeli
   #   POST /api/sentiment          -> body {"text":"..."} -> {label, probabilities}
   #   POST /api/cluster            -> body {"features":{...}} -> {cluster_id, n_clusters}
   #   GET  /api/recommend/{uid}    -> top-N rekomendacji; ?n=10&model=svd|knn_cosine|knn_pearson
   #   GET  /api/forecast/{genre}   -> prognoza next-quarter dla gatunku
   #   GET  /api/genres             -> lista dostepnych gatunkow
   #   GET  /api/users/sample       -> przykladowe user_ids w indeksie recsys
   # UI ma 4 zakladki: Sentyment, Rekomendacje, Prognoza, Profil->Klaster.
   # Health-bar na gorze pokazuje co jest gotowe (sentiment / cluster / recsys / popularity).

== Sprawdzenie poprawnosci wynikow ==

- data/processed/users_1m.parquet      -> 6040 wierszy, brak NaN w kolumnach demograficznych
- data/processed/ratings_1m.parquet    -> 1_000_209 wierszy, ratingi w {1,2,3,4,5}
- data/processed/ratings_20m.parquet   -> 20_000_263 wiersze, ratingi w [0.5, 5.0]
- reports/missing_report.csv           -> istnieje, lista kolumn z % brakow
- reports/k_selection.csv              -> 14 wierszy (k od 2 do 15), wybrane k* z najwyzszym silhouette
- silhouette_score dla wybranego k*    -> > 0.10 (realistyczny prog dla mieszanych cech)
- models/kmeans.joblib + scaler.joblib -> laduja sie przez joblib.load,
                                          predict na nowym profilu zwraca cluster_id w [0, k*-1]
- models/sentiment_clf.joblib          -> accuracy=0.99 na zbiorze testowym
- models/sentiment_vectorizer.joblib   -> TF-IDF, 5000 terms
- reports/recsys/metrics.csv           -> 3 wiersze (svd, knn_cosine, knn_pearson),
                                         RMSE w 0.85-1.05, MAE w 0.65-0.85
- reports/recsys/similarity_comparison.csv -> cosine vs pearson, jeden zwykle wygrywa min. 1 metryke
- models/recsys/svd.joblib + item_knn_*.joblib -> laduja sie, top_n_for_user(user_id=1, n=10)
                                                  zwraca 10 slownikow z movie_id+title+predicted_rating
- reports/popularity/model_comparison.csv  -> 3 wiersze (linear/ridge/random_forest), rmse_log_test < ~0.4
- reports/popularity/forecast_next.csv     -> ~20 wierszy (po gatunku), predicted_n_ratings > 0
- models/popularity/best_model.joblib      -> laduje sie, forecast_next_quarter("Drama") zwraca dict
- reports/bonus/outliers_pca.png           -> scatter PCA z ~5% punktow w kolorze czerwonym
- reports/bonus/mutual_info_top20.csv      -> 20 wierszy, wartosci mutual_info > 0 dla top cech
- GET http://127.0.0.1:8000/api/health     -> {"ready":{"sentiment":true,"cluster":true,"recsys":true,"popularity":true}}

== Klasyfikacja nowego profilu uzytkownika do klastra (uzycie modelu) ==

  import joblib, pandas as pd
  scaler = joblib.load("models/scaler.joblib")
  kmeans = joblib.load("models/kmeans.joblib")
  feature_cols = joblib.load("models/feature_cols.joblib")
  new_profile = pd.DataFrame([{...}])[feature_cols]
  cluster_id = int(kmeans.predict(scaler.transform(new_profile))[0])
