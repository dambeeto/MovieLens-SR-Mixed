# MovieLens-SR-Mixed

End-to-end data exploration project on the MovieLens 1M and 20M datasets. It covers data preparation, four machine learning categories (clustering, classification, recommendation, regression), and a small web application that exposes every trained model through a REST API and a single-page UI.

---
## Overview

Four independent components, one shared data pipeline:

1. **User profiling (KMeans)** on MovieLens 1M, using genre preferences combined with demographic features.
2. **Comment sentiment classification** (TF-IDF + Logistic Regression) on MovieLens 20M tags, with weak labels from VADER after a part-of-speech filter.
3. **Movie recommendations** (collaborative filtering: SVD plus item-based KNN with cosine and Pearson similarity) on MovieLens 1M, with a cold-start fallback for profiles the model has not seen.
4. **Genre popularity forecasting** (regression with seasonal and autoregressive features) on quarterly aggregates from MovieLens 20M.

Two bonus modules:

- **Outlier detection** on user feature vectors using IsolationForest.
- **Feature selection** diagnostics using VarianceThreshold and mutual information versus cluster id.

A FastAPI application wraps all four components into HTTP endpoints and a small HTML+JavaScript demo page.

---

## Data

The pipeline downloads two MovieLens releases automatically:

| Dataset | What it contains | Used for |
|---|---|---|
| **MovieLens 1M** | `users.dat` (gender, age, occupation, zip), `movies.dat` (title, year, genres), `ratings.dat` (1-5, timestamp). About 6 040 users and 1 000 209 ratings. | demographics, clustering, recommendation |
| **MovieLens 20M** | `ratings.csv` (0.5-5.0), `tags.csv` (free-text tags), `genome-tags.csv`, `genome-scores.csv`. About 138 000 users and 20 million ratings, but no demographics. | sentiment classification (tags), popularity forecasting (ratings volume per quarter) |

The two datasets are intentionally **not joined on `user_id`**, because they cover different user populations. Each iteration picks the dataset that best fits its task.

## Full pipeline

Every script below is idempotent. Re-running a step skips work that already produced its output, unless an artefact is missing.

### 1. Download the datasets

```powershell
python scripts/01_download.py
```

Downloads `ml-1m.zip` (fast) and `ml-20m.zip` (about 5 minutes depending on bandwidth), extracts both into `data/raw/`. Skips files that already exist.

### 2. Clean, identify missing values, split

```powershell
python scripts/02_clean.py
```

Output:

- `data/processed/users_1m.parquet`, `movies_1m.parquet`, `ratings_1m.parquet`
- `data/processed/ratings_20m.parquet`, `tags_20m.parquet`
- `data/processed/train.parquet`, `test.parquet` (chronological 80/20 split on ml-1m ratings)
- `reports/missing_report.csv` (per-column missingness for every cleaned table)

### 3. Exploratory plots

```powershell
python scripts/03_eda.py
```

Output:

- `reports/ratings_distribution.png`
- `reports/users_demographics.png`
- `reports/genres_popularity.png`
- `reports/ratings_over_time.png`

### 4. Build user features

```powershell
python scripts/04_features.py
```

Output:

- `data/processed/user_features.parquet` (about 6 040 users by about 59 features: 18 mean ratings per genre, 18 rating share per genre, age midpoint, female flag, 21 one-hot occupations)

### 5. Train KMeans and interpret clusters

```powershell
python scripts/05_cluster.py
```

Sweeps `k` in [2, 16], picks the best `k` by silhouette score with a Davies-Bouldin tie-breaker. Output:

- `models/scaler.joblib`, `kmeans.joblib`, `feature_cols.joblib`
- `data/processed/user_clusters.parquet` (user_id to cluster_id)
- `reports/k_selection.csv` (silhouette and Davies-Bouldin for every k tested)
- `reports/clusters_pca.png`
- `reports/cluster_profiles.csv` (top 5 genres, demographics per cluster)
- Console log with a short description of each cluster

### 6. Sentiment classification on tags

```powershell
python scripts/06_sentiment.py
```

Steps inside the script:

1. Filter `tags_20m.parquet` to tags that contain adjectives or adverbs (NLTK part-of-speech tagger). Removes meta tags such as "based on novel" or "imdb top 250".
2. Label the remaining tags with VADER. Drop neutrals (compound in [-0.05, +0.05]). Binary positive or negative labels.
3. Train TF-IDF (word 1-2 plus char_wb 3-5, FeatureUnion, 5 000 terms) and Logistic Regression with stratified 80/20 split.

Output:

- `models/sentiment_vectorizer.joblib`, `sentiment_clf.joblib`
- Classification reports printed to the console

### 7. Recommender system

```powershell
python scripts/07_recsys.py
```

Trains three models on `train.parquet` and evaluates on `test.parquet`:

- SVD with 50 latent factors using `scipy.sparse.linalg.svds` on a user-mean-centered matrix
- Item-based KNN with cosine similarity, 30 neighbours
- Item-based KNN with Pearson similarity (centered cosine), 30 neighbours

Output:

- `models/recsys/svd.joblib`, `item_knn_cosine.joblib`, `item_knn_pearson.joblib`
- `models/recsys/user_index.joblib`, `item_index.joblib`
- `reports/recsys/metrics.csv` (RMSE, MAE, Hit-rate@10 per model)
- `reports/recsys/similarity_comparison.csv`
- `reports/recsys/top10_examples.csv`
- `reports/recsys/rating_predictions_scatter.png`

### 8. Genre popularity forecasting

```powershell
python scripts/08_popularity.py
```

Pipeline:

1. Aggregate `ratings_20m.parquet` and `ml-20m/movies.csv` into a dense grid of (genre, quarter).
2. Drop the last quarter if it is suspiciously low (ml-20m ends mid Q1 2015).
3. Build features: `log1p(n_ratings)` as the target, calendar (year_idx, sin and cos of quarter), lags 1, 2, 4, and a 4-quarter rolling mean.
4. Walk-forward evaluation with the last 4 quarters held out. Compares LinearRegression, Ridge, and RandomForestRegressor; picks the best by RMSE-log on the held-out set.
5. Forecast the next quarter for every genre.

Output:

- `models/popularity/best_model.joblib`, `history.parquet`, `meta.joblib`
- `reports/popularity/model_comparison.csv`
- `reports/popularity/forecast_next.csv`
- `reports/popularity/time_series_per_genre.png`

### 9. Bonus

```powershell
python scripts/09_bonus.py
```

Runs two analyses on `user_features.parquet`:

- IsolationForest with contamination 0.05. Output: `data/processed/user_outliers.parquet`, `models/bonus/isolation_forest.joblib`, `reports/bonus/outliers_pca.png`, `reports/bonus/top_anomalous_users.csv`.
- VarianceThreshold report at 0.01 plus mutual information ranking versus `cluster_id` from `user_clusters.parquet`. Output: `reports/bonus/variance_report.csv`, `reports/bonus/mutual_info_top20.csv`.

### 10. Run the web application

```powershell
uvicorn main:app --reload
```

By default the application listens on `http://127.0.0.1:8000`. Open that URL in a browser to use the demo UI.

## API reference

| Method | Path | Description |
|---|---|---|
| GET | `/` | Serves the single-page UI (`static/index.html`) |
| GET | `/api/health` | Returns availability flags for every artefact: `sentiment`, `cluster`, `recsys`, `popularity` |
| POST | `/api/sentiment` | Classify a free-text comment. Body: `{"text": "..."}` |
| POST | `/api/cluster` | Assign a user profile vector to a cluster. Body: `{"features": {"name": value, ...}}`. Missing features default to 0 |
| GET | `/api/cluster/profile/{cluster_id}` | Human-readable description of a cluster: size, typical demographics, and top genres (from `cluster_profiles.csv`) |
| GET | `/api/cluster/recommend/{cluster_id}` | Cold-start picks for a profile: the most-watched movies among that cluster's members |
| GET | `/api/recommend/{user_id}` | Top-N recommendations for a user, plus the user's profile (gender, age, occupation, favourite genres). An unknown user returns `found: false` and a popularity-based cold-start list. Query parameters: `n` (default 10), `model` (`svd`, `knn_cosine`, or `knn_pearson`) |
| GET | `/api/forecast/{genre}` | Next-quarter popularity forecast for the given genre, plus the last quarter's actual rating count and the percentage trend |
| GET | `/api/genres` | List of genres available in the popularity model |
| GET | `/api/users/sample` | Sample user ids that exist in the recommender index (helps the UI demo) |
| GET | `/api/cluster/template` | Feature names plus per-feature training means; the UI uses the means to seed unselected genres with a neutral baseline |

All endpoints return JSON. Endpoints that depend on an artefact respond with HTTP 503 and a helpful message if the corresponding `joblib` file is missing.

## Usage examples

### Predict sentiment of a comment

```python
from src.sentiment.predict import predict_sentiment

predict_sentiment(["this movie was amazing", "boring and predictable"])
# [
#   {"text": "this movie was amazing", "label": "positive", "probabilities": {...}},
#   {"text": "boring and predictable", "label": "negative", "probabilities": {...}},
# ]
```

### Generate recommendations

```python
from src.recsys.predict import top_n_for_user

top_n_for_user(user_id=1000, n=10, model_name="svd")
# [
#   {"movie_id": 1196, "title": "Star Wars: Episode V - The Empire Strikes Back",
#    "year": 1980, "predicted_rating": 4.78},
#   ...
# ]
# model_name can also be "knn_cosine" or "knn_pearson".
# Movies already rated by the user in the training split are excluded.
# The recommender covers users in the training split (ids from about 635 to 6040);
# an id outside that range returns [] here and triggers the cold-start fallback in the API.
```

### Cold-start recommendations

```python
from src.recsys.cold_start import popular_top_n, recommend_for_cluster

# Unknown user: fall back to the globally most-watched movies.
popular_top_n(n=10)

# Brand-new profile assigned to cluster 4: recommend what that cluster's members watch most.
recommend_for_cluster(cluster_id=4, n=10)
# Both return [{"movie_id", "title", "year", "predicted_rating"}, ...] sorted by
# number of ratings, then by mean rating. predicted_rating is the average rating.
```

### Forecast the next quarter for a genre

```python
from src.popularity.predict import forecast_next_quarter

forecast_next_quarter("Drama")
# {
#   "genre": "Drama",
#   "next_period": "2015-06-30",
#   "predicted_log": 11.4997,
#   "predicted_n_ratings": 98687,
#   "last_period": "2015-03-31",
#   "last_quarter_n_ratings": 129444,
#   "trend_pct": -23.8,
#   "based_on_model": "random_forest",
# }
```

### Assign a new user profile to a cluster

```python
import joblib
import pandas as pd

scaler = joblib.load("models/scaler.joblib")
kmeans = joblib.load("models/kmeans.joblib")
feature_cols = joblib.load("models/feature_cols.joblib")

new_profile = pd.DataFrame([{"age_midpoint": 28, "is_female": 1, "rmean_Drama": 4.5}])
new_profile = new_profile.reindex(columns=feature_cols, fill_value=0.0)
cluster_id = int(kmeans.predict(scaler.transform(new_profile.to_numpy()))[0])
```

## Sanity checks for a fresh run

After steps 1 to 9 you should see roughly these numbers:

- `data/processed/users_1m.parquet` has 6 040 rows, no missing values in the demographic columns
- `data/processed/ratings_1m.parquet` has 1 000 209 rows with ratings in {1, 2, 3, 4, 5}
- `data/processed/ratings_20m.parquet` has 20 000 263 rows with ratings in [0.5, 5.0]
- `reports/missing_report.csv` exists and lists every column with its missing percentage
- `reports/k_selection.csv` has rows for every k tested; the chosen `k*` has the highest silhouette
- Silhouette score for the chosen `k*` is above about 0.10 (a realistic threshold for mixed feature types)
- `models/kmeans.joblib` and `scaler.joblib` load via `joblib.load` and predict a cluster id in `[0, k*-1]` for a new profile
- `data/processed/tags_labeled.parquet` has tens of thousands of rows with `tag`, `sentiment`, `vader_compound`
- `models/recsys/svd.joblib` and `item_knn_*.joblib` load and `top_n_for_user(user_id=1000, n=10)` returns 10 dictionaries (the recommender covers training-split users, roughly ids 635 to 6040)
- `reports/popularity/model_comparison.csv` has 3 rows with `rmse_log_test` below about 0.4
- `reports/popularity/forecast_next.csv` has one row per genre with `predicted_n_ratings` greater than 0
- `models/popularity/best_model.joblib` loads and `forecast_next_quarter("Drama")` returns a dictionary
- `reports/bonus/outliers_pca.png` exists; about 5 percent of users are highlighted
- `reports/bonus/mutual_info_top20.csv` has 20 rows with non-zero mutual information for the top features
- `GET http://127.0.0.1:8000/api/health` returns `{"ready": {"sentiment": true, "cluster": true, "recsys": true, "popularity": true}}`
